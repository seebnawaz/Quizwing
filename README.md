# QuizWing Content Repo

This repo is the **single source of truth for all MCQ content** used by the QuizWing mobile app.
The app reads JSON files directly from `raw.githubusercontent.com/<user>/quizwing-content/main/...`
and caches them locally. Every commit = an instant content update for all app users.

> **Important:** This repo is *separate* from the QuizWing WordPress site.
> WordPress = web SEO + marketing. This repo = mobile app content.
> They are intentionally decoupled.

## How to add or edit content

### Option 1 — On a phone or any browser
1. Navigate to the file on GitHub.com
2. Click the pencil icon (✏️ Edit this file)
3. Make changes
4. Scroll down → **Commit changes** → done
5. App users see the update within minutes

### Option 2 — From your laptop
```bash
git clone https://github.com/YOUR-USERNAME/quizwing-content.git
cd quizwing-content
# Edit any JSON file in content/
git add . && git commit -m "Added 20 new planet MCQs" && git push
```

### Option 3 — AI-assisted (recommended for bulk)
1. Ask Claude/ChatGPT to "write 20 MCQs about [topic] in QuizWing JSON format"
2. Paste the output into the relevant `.json` file
3. Commit

## MCQ JSON Schema

Every topic file follows this exact structure:

```json
{
  "topic_id": "planets",
  "topic_name": "Planets",
  "subject": "everyday-science",
  "version": 2,
  "updated_at": "2026-05-04",
  "description": "Solved MCQs on Mercury, Venus, Earth, Mars, Jupiter, Saturn, Uranus, Neptune.",
  "icon": "🪐",
  "mcqs": [
    {
      "id": "pln-001",
      "q": "Which is the largest planet in the solar system?",
      "options": ["Saturn", "Jupiter", "Earth", "Neptune"],
      "correct": 1,
      "explanation": "Jupiter — eleven Earths could fit across Jupiter's equator. Gas giant made of hydrogen and helium.",
      "tags": ["jupiter", "largest", "anchor-fact"],
      "difficulty": "easy"
    }
  ]
}
```

### Field reference

| Field | Type | Required | Notes |
|---|---|---|---|
| `topic_id` | string | yes | Slug, must match folder/file name |
| `topic_name` | string | yes | Display name |
| `subject` | string | yes | Parent subject slug |
| `version` | int | yes | **Bump on every change** — app uses this to detect updates |
| `updated_at` | date | yes | YYYY-MM-DD |
| `description` | string | yes | One-line topic summary |
| `icon` | string | optional | Emoji for the topic |
| `mcqs[].id` | string | yes | Unique ID, e.g. `pln-001` (prefix = topic) |
| `mcqs[].q` | string | yes | The question text |
| `mcqs[].options` | array[4] | yes | Exactly 4 answer choices |
| `mcqs[].correct` | int | yes | Index of correct option (0–3) |
| `mcqs[].explanation` | string | yes | Why the answer is correct |
| `mcqs[].tags` | array[string] | optional | For search and filtering |
| `mcqs[].difficulty` | enum | optional | "easy", "medium", "hard" |

## The `manifest.json` (auto-generated)

This file is **generated by a GitHub Action** — you never edit it manually. It lists every topic file with its current version, so the app can fetch only what changed.

```json
{
  "schema_version": 1,
  "generated_at": "2026-05-04T12:00:00Z",
  "subjects": [
    {
      "id": "everyday-science",
      "name": "Everyday Science",
      "icon": "🔬",
      "order": 1,
      "topics": [
        { "id": "planets",     "version": 2, "mcq_count": 55, "url": "content/everyday-science/planets.json" },
        { "id": "galaxies",    "version": 1, "mcq_count": 42, "url": "content/everyday-science/galaxies.json" }
      ]
    }
  ]
}
```

---

## Versioning rules (important)

Every time you edit a topic file:
1. **Bump the `version` field by 1**
2. **Update `updated_at`** to today's date

The Flutter app compares versions to know which files to re-download. Without bumping, the app won't fetch your changes.

A GitHub Action automatically regenerates `manifest.json` on every push — no manual work.

## What lives here ✅

- MCQ JSON files
- Subject metadata
- Topic illustrations / SVG icons
- Static educational content (definitions, quick facts)
- App-wide constants (Pro pricing labels, etc.)

## What does NOT live here ❌

- User accounts (→ Firebase Auth)
- Quiz attempt history (→ Firestore)
- Pro subscriptions (→ Firestore + Google Play Billing)
- Push notifications (→ Firebase Cloud Messaging)
- Anything private/secret
